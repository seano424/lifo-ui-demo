  3. CRUD Actions Built (from hooks)

  Products CRUD (use-products.ts):
  - Create: createProduct() with store-aware validation
  - Read: useProducts(), useProduct() with infinite scroll
  - Update: updateProduct(), price updates, category changes, toggle active status
  - Delete: deleteProduct() with cache invalidation
  - Filtering: by category, expiring products

  Batches CRUD (use-batches.ts):
  - Create: createBatch() with store validation
  - Read: useBatches(), useBatch(), useBatchesForProduct()
  - Update: quantity, price, location, status changes (expired/damaged/sold-out)
  - Delete: deleteBatch() with product sync
  - Special: process sales, reserve quantities, batch alerts for expiring/low stock

  Store Users CRUD (use-store-users.ts):
  - Create: addUserToStore() with role assignment
  - Read: useStoreUsers(), useStoreUser() by role/status
  - Update: role changes, permissions, PIN auth settings
  - Delete: removeUserFromStore()
  - Filtering: by role (owner/manager/employee), active status, PIN enabled

  Users Management (use-users.ts):
  - Read: useUsers(), useCurrentUser() with metadata
  - Update: profile, phone, language preference, PIN settings
  - Special: role checking, permission validation, PIN lock management
  - Language/phone support with dedicated update mutations

  4. User Management Setup

  Authentication System:
  - Supabase Auth integration with email/password
  - PIN-based authentication for quick access
  - Password reset flow with email verification
  - Session management with JWT tokens

  Role-Based Access Control:
  - Store-level roles: Owner, Manager, Employee, Staff
  - Global roles mapped from store roles (admin, manager, employee)
  - Fine-grained permissions system (can_manage_users, can_view_analytics, etc.)
  - Store-aware context for multi-tenant support

  User Features:
  - Profile management with avatar, full name, username
  - Language preference support (internationalization)
  - Phone number management for SMS notifications
  - PIN authentication with attempts tracking and lockout
  - Activity tracking (last login, created/updated timestamps)

  Permission System:
  - Store-specific permissions
  - PIN access levels (basic, elevated, admin)
  - Context-aware permission checking (can() function)
  - Special permissions: manage settings, upload inventory, apply discounts

  Security Features:
  - PIN lockout after failed attempts
  - PIN expiration and reset capabilities
  - Email and phone verification status
  - Active/inactive user status per store
  - Store isolation (users only see their store's data)

  This represents a comprehensive inventory management system with multi-store support, robust user management, and full CRUD operations for products,
  batches, and team members.
